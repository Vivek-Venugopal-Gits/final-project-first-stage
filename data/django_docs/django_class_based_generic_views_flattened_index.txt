SECTION: Class-Based Generic Views - Flattened Index
PURPOSE: This section provides an alternate organization of the reference documentation for class-based views, listing effective attributes and methods under each view.
CONCEPT: For each view, the effective attributes and methods from the class tree are represented.
SECTION: Simple Generic Views
PURPOSE: This section describes base views that provide foundational functionality for rendering templates or redirects.
SECTION: View
PURPOSE: The base view class from which all other class-based views inherit.
CONCEPT: Not strictly a generic view; can be imported from django.views.
CONCEPT: Ancestors (MRO): django.views.generic.base.View
Attributes:

ATTRIBUTEDESCRIPTIONhttp_method_namesThe list of HTTP method names that this view will accept. Default: ["get", "post", "put", "patch", "delete", "head", "options", "trace"]
Methods:

as_view(initkwargs): Returns a callable view that takes a request and returns a response. The returned view has view_class and view_initkwargs attributes. When called, setup() assigns the HttpRequest to request, and URL-captured arguments to args and kwargs. Then dispatch() is called. If the view defines asynchronous (async def) handlers, as_view() marks the callable as a coroutine. Raises ImproperlyConfigured if both async and sync handlers are defined.
setup(request, args, kwargs): Performs initialization before dispatch(). Assigns request to the view’s request attribute, and URL-captured arguments to args and kwargs. Must call super() if overridden.
dispatch(request, args, kwargs): The core method that handles the request and returns an HTTP response. Inspects the HTTP method and delegates to methods like get(), post(), etc. HEAD requests are delegated to get() by default. Override head() if different handling is needed.
http_method_not_allowed(request, args, kwargs): Called when an unsupported HTTP method is used. Returns HttpResponseNotAllowed with a plain-text list of allowed methods.
options(request, args, kwargs): Handles OPTIONS requests. Returns a response with an Allow header listing allowed HTTP methods. If other HTTP method handlers are async, the response is wrapped in a coroutine.

SECTION: TemplateView
PURPOSE: Renders a given template, with the context containing parameters captured in the URL.
CONCEPT: Ancestors (MRO): django.views.generic.base.TemplateResponseMixin, django.views.generic.base.ContextMixin, django.views.generic.base.View
CONCEPT: Context populated via ContextMixin with keyword arguments captured from the URL pattern. Can also include context via the extra_context keyword argument in as_view().
Attributes:

ATTRIBUTEDESCRIPTIONcontent_typeThe content type to use for the response. Passed as a keyword argument to response_class. Default: None (uses 'text/html').extra_contextA dictionary to include in the context.http_method_namesThe list of HTTP method names that this view will accept. Default: ["get", "post", "put", "patch", "delete", "head", "options", "trace"]response_classThe response class to be returned by render_to_response method. Default: TemplateResponse.template_engineThe NAME of a template engine to use for loading the template. Passed as the using keyword argument to response_class. Default: None (searches all configured engines).template_nameThe full name of a template to use as defined by a string. Not defining a template_name will raise ImproperlyConfigured.
Methods:

as_view(): Returns a callable view that takes a request and returns a response.
dispatch(): The core method that handles the request and returns an HTTP response.
get(): Renders the template.
get_context_data(**kwargs): Returns a dictionary representing the template context. The keyword arguments provided will make up the returned context.
head(): Delegates to get().
http_method_not_allowed(): Returns HttpResponseNotAllowed with a list of allowed methods.
render_to_response(context, **response_kwargs): Returns a self.response_class instance. If keyword arguments are provided, they will be passed to the constructor of the response class. Calls get_template_names() to obtain the list of template names.
setup(): Performs initialization before dispatch().
get_template_names(): Returns a list of template names to search for when rendering the template. The default implementation will return a list containing template_name (if specified).

SECTION: RedirectView
PURPOSE: Redirects to a given URL.
CONCEPT: The URL may contain dictionary-style string formatting, interpolated against URL-captured parameters. Any literal "%" in the URL must be written as "%%". If url is None, returns HttpResponseGone (410).
CONCEPT: Ancestors (MRO): django.views.generic.base.View
Attributes:

ATTRIBUTEDESCRIPTIONhttp_method_namesThe list of HTTP method names that this view will accept. Default: ["get", "post", "put", "patch", "delete", "head", "options", "trace"]pattern_nameThe name of the URL pattern to redirect to. Reverse lookup uses the same args and kwargs passed to the view.permanentWhether the redirect is permanent. True: uses HTTP 301. False: uses HTTP 302 (default).query_stringWhether to pass the GET query string to the new location. True: appends query string. False: discards it (default).urlThe URL to redirect to, as a string. Or None to raise a 410 (Gone) error.
Methods:

as_view(): Returns a callable view that takes a request and returns a response.
dispatch(): The core method that handles the request and returns an HTTP response.
delete(): Calls get_redirect_url().
get(): Calls get_redirect_url().
head(): Calls get_redirect_url().
http_method_not_allowed(): Returns HttpResponseNotAllowed with a list of allowed methods.
options(): Calls get_redirect_url().
post(): Calls get_redirect_url().
put(): Calls get_redirect_url().
setup(): Performs initialization before dispatch().
get_redirect_url(*args, **kwargs): Constructs the target URL for redirection. Uses url and performs % parameter expansion based on captured URL groups. If url is not set, attempts to reverse pattern_name using captured args and kwargs. If query_string is True, appends the query string. Subclasses can override for custom behavior, but must return a redirect-ready URL string.

from django.shortcuts import get_object_or_404
from django.views.generic.base import RedirectView
from articles.models import Article

class ArticleCounterRedirectView(RedirectView):
    permanent = False
    query_string = True
    pattern_name = "article-detail"

    def get_redirect_url(self, *args, **kwargs):
        article = get_object_or_404(Article, pk=kwargs["pk"])
        article.update_counter()
        return super().get_redirect_url(*args, **kwargs)

SECTION: DetailView
PURPOSE: Displays a single object.
CONCEPT: Key attribute: self.object contains the object being displayed.
CONCEPT: Ancestors (MRO): SingleObjectTemplateResponseMixin, TemplateResponseMixin, BaseDetailView, SingleObjectMixin, View
Attributes:

ATTRIBUTEDESCRIPTIONcontent_typeThe content type to use for the response. Passed as a keyword argument to response_class. By default, pk_url_kwarg is 'pk'.query_pk_and_slugIf True, causes get_object() to perform its lookup using both the primary key and the slug. Defaults to False. This attribute can help mitigate insecure direct object reference attacks.querysetA QuerySet that represents the objects. If provided, the value of queryset supersedes the value provided for model. Warning: queryset is a class attribute with a mutable value so care must be taken when using it directly. Before using it, either call its all() method or retrieve it with get_queryset() which takes care of the cloning behind the scenes.response_classThe response class to be returned by render_to_response method. Default: TemplateResponse.slug_fieldThe name of the field on the model that contains the slug. By default, slug_field is 'slug'.slug_url_kwargThe name of the URLConf keyword argument that contains the slug. By default, slug_url_kwarg is 'slug'.template_engineThe NAME of a template engine to use for loading the template. Passed as the using keyword argument to response_class. Default: None (searches all configured engines).template_nameThe full name of a template to use as defined by a string. Not defining a template_name will raise ImproperlyConfigured.template_name_fieldThe field on the current object instance that can be used to determine the name of a candidate template. If either template_name_field itself or the value of the template_name_field on the current object instance is None, the object will not be used for a candidate template name.template_name_suffixThe suffix to append to the auto-generated candidate template name. Default: '_detail'.
Methods:

as_view(): Returns a callable view that takes a request and returns a response.
dispatch(): The core method that handles the request and returns an HTTP response.
http_method_not_allowed(): Returns HttpResponseNotAllowed with a list of allowed methods.
get_template_names(): Returns a list of candidate template names. Returns a list containing template_name, if set. Otherwise, returns a list containing: the contents of the template_name_field field on the object instance that the view is operating upon (if available), <app_label>/<model_name><template_name_suffix>.html
get_slug_field(): Returns the name of a slug field to be used to look up by slug. By default this returns the value of slug_field.
get_queryset(): Returns the queryset that will be used to retrieve the object that this view will display. By default, get_queryset() returns the value of the queryset attribute if it is set, otherwise it constructs a QuerySet by calling the all() method on the model attribute’s default manager.
get_object(queryset=None): Returns the single object that this view will display. If queryset is provided, that queryset will be used as the source of objects; otherwise, get_queryset() will be used. get_object() looks for a pk_url_kwarg argument in the arguments to the view; if this argument is found, this method performs a primary-key based lookup using that value. If this argument is not found, it looks for a slug_url_kwarg argument, and performs a slug lookup using the slug_field. When query_pk_and_slug is True, get_object() will perform its lookup using both the primary key and the slug.
get_context_object_name(obj): Return the context variable name that will be used to contain the data that this view is manipulating. If context_object_name is not set, the context name will be constructed from the model_name of the model that the queryset is composed from. For example, the model Article would have context object named 'article'.
get_context_data(**kwargs): Returns context data for displaying the object. The base implementation of this method requires that the self.object attribute be set by the view (even if None). Be sure to do this if you are using this mixin without one of the built-in views that does so. It returns a dictionary with these contents: object: The object that this view is displaying (self.object). context_object_name: self.object will also be stored under the name returned by get_context_object_name(), which defaults to the lowercased version of the model name. Context variables override values from template context processors. Any variables from get_context_data() take precedence over context variables from context processors. For example, if your view sets the model attribute to User, the default context object name of user would override the user variable from the django.contrib.auth.context_processors.auth() context processor. Use get_context_object_name() to avoid a clash.
get(): Adds object to context.
render_to_response(context, **response_kwargs): Returns a self.response_class instance. If keyword arguments are provided, they will be passed to the constructor of the response class. Calls get_template_names() to obtain the list of template names.
setup(): Performs initialization before dispatch().
