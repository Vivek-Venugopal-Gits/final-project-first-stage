APPLICABILITY: All Django projects and deployments
SECTION: CORE SETTINGS OVERRIDE WARNING: Preserve required components when overriding non-empty list or dictionary defaults like STATICFILES_FINDERS.
CONCEPT: ABSOLUTE_URL_OVERRIDES
Default: {} (Empty dictionary)

Dictionary mapping "app_label.model_name" strings to functions that take a model object and return its URL. Overrides get_absolute_url() methods per installation.

CODE:
ABSOLUTE_URL_OVERRIDES = {
    "blogs.blog": lambda o: "/blogs/%s/" % o.slug,
    "news.story": lambda o: "/stories/%s/%s/" % (o.pub_year, o.slug),
}
Model name uses lowercase regardless of actual model class name case.

CONCEPT: ADMINS
Default: [] (Empty list)

List of email addresses receiving code error notifications when DEBUG=False and AdminEmailHandler configured in LOGGING. Django emails exception details from request/response cycle.

CODE:
ADMINS = ["john@example.com", '"Ng, Mary" <mary@example.com>']
Changed in Django 6.0: Older versions required (name, address) tuples.

CONCEPT: ALLOWED_HOSTS
Default: [] (Empty list)

List of strings representing host/domain names Django site can serve. Prevents HTTP Host header attacks.

Fully qualified names match Host header exactly (case-insensitive, excluding port)

Values starting with period act as subdomain wildcards ('.example.com' matches all subdomains)

'*' matches anything; requires custom Host header validation in first MIDDLEWARE position

When DEBUG=True and ALLOWED_HOSTS empty, validates against ['.localhost', '127.0.0.1', '[::1]']. Checked during tests and get_host() calls. Direct request.META['Host'] access bypasses validation.

CONCEPT: APPEND_SLASH
Default: True

When True, redirects non-matching URLconf patterns without trailing slash to same URL with slash appended. Only active with CommonMiddleware. POST data lost on redirect. See PREPEND_WWW.

SECTION: CACHES CONFIGURATION: Defines all cache backends with default configuration.
Default:

CODE:
{
    "default": {
        "BACKEND": "django.core.cache.backends.locmem.LocMemCache",
    }
}
Nested dictionary mapping cache aliases to individual cache options. Must define 'default' cache.

Built-in backends:

'django.core.cache.backends.db.DatabaseCache'

'django.core.cache.backends.dummy.DummyCache'

'django.core.cache.backends.filebased.FileBasedCache'

'django.core.cache.backends.locmem.LocMemCache'

'django.core.cache.backends.memcached.PyLibMCCache'

'django.core.cache.backends.redis.RedisCache'

Use fully-qualified path for custom backends.

KEY_FUNCTION Dotted path to function composing prefix, version, and key into final cache key.

KEY_PREFIX Default: '' Prepended to all cache keys.

LOCATION Cache location (directory, host:port, etc.).

OPTIONS Backend-specific parameters.

TIMEOUT Default: 300 Cache entry expiration in seconds. None prevents expiration; 0 disables caching.

VERSION Default: 1 Default version for cache keys.

CACHES requires 'default' alias

LocMemCache used by default

Backend-specific options vary

SECTION: CSRF PROTECTION SETTINGS: Controls Cross Site Request Forgery protection behavior.
CSRF_COOKIE_AGE Default: 31449600 (1 year)

CSRF cookie age in seconds. Long expiration avoids browser cache/form submission issues.

CSRF_COOKIE_DOMAIN Default: None

Domain for CSRF cookie. Use ".example.com" for cross-subdomain POST requests.

CSRF_COOKIE_HTTPONLY Default: False

Sets HttpOnly flag on CSRF cookie. Prevents JavaScript access (minimal CSRF benefit).

CSRF_COOKIE_NAME Default: 'csrftoken'

Name of CSRF authentication token cookie.

CSRF_COOKIE_PATH Default: '/'

Path for CSRF cookie. Matches Django installation URL path.

CSRF_COOKIE_SAMESITE Default: 'Lax'

SameSite flag value for CSRF cookie.

CSRF_COOKIE_SECURE Default: False

Secure cookie flag for CSRF cookie (HTTPS only).

CSRF_USE_SESSIONS Default: False

Store CSRF token in session instead of cookie. Requires django.contrib.sessions and SessionMiddleware before exception-raising middleware.

CSRF_FAILURE_VIEW Default: 'django.views.csrf.csrf_failure'

Dotted path to view handling CSRF failures. Signature: def csrf_failure(request, reason="").

CSRF_HEADER_NAME Default: 'HTTP_X_CSRFTOKEN'

Request header name for CSRF authentication (normalized from client header).

CSRF_TRUSTED_ORIGINS Default: []

Trusted origins for unsafe requests (POST). Supports subdomains with 'https://*.example.com'.

SUMMARY:
CSRF cookies valid for 1 year by default

Session storage alternative available

Trusted origins prevent cross-subdomain issues

HttpOnly provides minimal additional protection

SECTION: DATABASES CONFIGURATION: Complete database connection settings.
Default: {} (Empty dictionary)

Nested dictionary mapping aliases to database options. Must define 'default' database.

SQLite Example:

CODE:
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.sqlite3",
        "NAME": "mydatabase",
    }
}
PostgreSQL Example:
CODE:
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.postgresql",
        "NAME": "mydatabase",
        "USER": "mydatabaseuser",
        "PASSWORD": "mypassword",
        "HOST": "127.0.0.1",
        "PORT": "5432",
    }
}
DATABASE OPTIONS
ATOMIC_REQUESTS Default: False

Wraps each view in transaction.

AUTOCOMMIT Default: True

Set False to disable Django transaction management.

ENGINE Database backend:

'django.db.backends.postgresql'

'django.db.backends.mysql'

'django.db.backends.sqlite3'

'django.db.backends.oracle'

HOST Default: '' (localhost). Unix socket for MySQL if starts with /.

NAME Database name (SQLite: full file path, use forward slashes).

CONN_MAX_AGE Default: 0 Connection lifetime in seconds. None for unlimited.

CONN_HEALTH_CHECKS Default: False

Health checks persistent connections before reuse.

OPTIONS Backend-specific connection parameters.

PASSWORD Database password.

PORT Default: '' (default port).

TIME_ZONE Default: None Database connection time zone.

DISABLE_SERVER_SIDE_CURSORS Default: False (PostgreSQL)

Disables server-side cursors with QuerySet.iterator().

USER Database username.

TEST DATABASE OPTIONS
CHARSET, COLLATION Backend-specific test database encoding.

DEPENDENCIES Creation order dependencies.

MIGRATE Default: True Run migrations on test database.

MIRROR Database to mirror during testing (TransactionTestCase only).

NAME Test database name ('test_' + DATABASE_NAME by default).

TEMPLATE PostgreSQL template database.

Oracle-specific: CREATE_DB, CREATE_USER, TBLSPACE, DATAFILE options.
